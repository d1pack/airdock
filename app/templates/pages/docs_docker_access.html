Доступ курьера к Docker

Почему контейнеры не отображаются

Airdock подключается к серверу по SSH под пользователем курьера и выполняет docker ps -a. Если у этого пользователя нет доступа к Docker, метрики сервера работают, но список контейнеров не приходит.

Вариант 1: добавить пользователя в группу docker

  1. Проверьте пользователя, который указан в курьере. 
    whoami
  2. Добавьте его в группу Docker. Замените develop на нужного пользователя. 
    sudo usermod -aG docker develop
  3. Перезайдите в SSH-сессию или перезапустите сервер, чтобы группа применилась. 
    exit
ssh develop@SERVER_IP
  4. Проверьте доступ без sudo. 
    docker ps -a

Вариант 2: разрешить sudo только для Docker

  1. Откройте sudoers через безопасный редактор. 
    sudo visudo
  2. Добавьте строку. Путь к Docker можно проверить командой which docker. 
    develop ALL=(root) NOPASSWD: /usr/bin/docker
  3. Проверьте, что команда работает без запроса пароля. 
    sudo -n docker ps -a

Что должно получиться

После настройки Airdock начнет получать список контейнеров автоматически. На странице проекта курьер будет светиться активным, а таблица контейнеров заполнится результатом docker ps -a.
